# SON-20 · 可视化：动力链对比图
> 来源：Multica issue SON-20 的智能体评论产出。**状态：待队长/QA 审核**（尚未逐条验证准确性）。

---

## 评论 1 · 智能体

已完成动力流对比图的结构方案与视觉规范，建议开发按“一张 4 行泳道图 + 模式切换/动效”的形式实现。

## 设计目标

让非汽车专业新人一眼看懂 4 个差异：

1. 能量从哪里来：燃油 / 电池 / 二者都有。
2. 谁最终驱动车轮：发动机 / 电机 / 二者共同。
3. 发动机是否和车轮有机械直连。
4. 为什么纯电通常不需要传统多挡变速箱，只需要电机后的单级减速/终传。

## 总体版式

采用 16:9 横向画布，桌面端推荐 1440 x 900；移动端改为四张可横滑卡片。

- 顶部：标题“能量如何到达车轮”，右侧放图例。
- 主体：四条横向泳道，从上到下依次为“燃油车 / 混动 / 增程 / 纯电”。
- 每条泳道统一按“能量源 -> 转换装置 -> 耦合/控制 -> 驱动装置 -> 车轮”从左到右排列。
- 每条泳道右侧放一句结论标签，避免用户只看部件不懂差异。

## 图例规则

- 橙色实线：燃油/发动机能量流。
- 蓝色实线：电能流。
- 深灰实线：机械动力流，表示实体轴/齿轮/传动连接。
- 青色虚线反向箭头：制动能量回收。
- 红色细箭头：热损失/尾气损失，可只在燃油和混动行弱化展示。
- 圆角矩形：部件节点；菱形或六边形：能量分流/耦合节点；圆形：车轮。

## 四种架构内容

### 1. 燃油车 ICE

主路径：

`燃油箱 -> 燃油泵/喷油 -> 发动机 ICE -> 离合器/液力变矩器 -> 多挡变速箱/CVT -> 差速器 -> 车轮`

视觉重点：

- 发动机到车轮是完整的机械传动链。
- 变速箱节点要突出，因为发动机高效转速区窄、低速扭矩不足，需要换挡/变速来适配车速。
- 无制动能量回收；制动时只用灰色虚线指向“摩擦制动热损失”。

右侧结论：

“发动机直接驱动车轮；变速箱负责把发动机转速/扭矩变成适合车速的输出。”

### 2. 混动 HEV/PHEV

建议画成主流“并联/功率分流混动”的抽象，不绑定某一品牌细节：

主路径：

`燃油箱 -> 发动机 ICE -> 动力耦合器/e-CVT/离合器 -> 差速器 -> 车轮`

电路径：

`电池 <-> 逆变器 <-> 电机/发电机 -> 动力耦合器/e-CVT/离合器`

回收路径：

`车轮 -> 电机/发电机 -> 逆变器 -> 电池`

视觉重点：

- 发动机和电机都可能参与驱动车轮，箭头可汇入同一个“动力耦合器”。
- 电池箭头必须是双向：既能放电助力，也能回收/发动机发电充电。
- 如果页面支持模式切换，混动行可切换“纯电低速 / 发动机巡航 / 加速助力 / 制动回收”四种状态。

右侧结论：

“发动机和电机共同管理效率：低速可电驱，加速可助力，制动可回收。”

### 3. 增程 EREV

主路径：

`燃油箱 -> 发动机 ICE -> 发电机 -> 逆变器/DC 母线 -> 驱动电机 -> 单级减速器 -> 车轮`

电池路径：

`电池 <-> 逆变器/DC 母线 -> 驱动电机`

回收路径：

`车轮 -> 驱动电机 -> 逆变器/DC 母线 -> 电池`

视觉重点：

- 发动机到车轮之间不要画机械连线；中间必须通过“发电机/DC 母线”转成电。
- 可在发动机与车轮之间放一个淡灰色断开的轴线并标注“无机械直连”，这是增程与普通混动最重要差异。
- 车轮永远由驱动电机带动。

右侧结论：

“发动机只发电，不直接驱动车轮；驾驶感更接近电动车。”

### 4. 纯电 BEV

主路径：

`动力电池 -> BMS/高压配电 -> 逆变器 -> 驱动电机 -> 单级减速器/差速器 -> 车轮`

充电支路：

`充电口/车载充电机 -> 动力电池`，可作为次级淡色支路，不抢主线。

回收路径：

`车轮 -> 驱动电机 -> 逆变器 -> 动力电池`

视觉重点：

- 不出现发动机、油箱、排气系统。
- “单级减速器”节点要和燃油车“多挡变速箱”做视觉对照：尺寸更小、层级更低。
- 旁边加简短解释：“电机低速即可输出大扭矩，且可在宽转速范围内高效工作，所以多数纯电车不需要传统多挡变速箱。”

右侧结论：

“电池供电，电机直接通过减速器驱动车轮；传动链最短。”

## 推荐交互

- 默认状态：展示“加速/驱动”能量流。
- 顶部用分段控件切换：`驱动`、`巡航`、`制动回收`。
- 切到“制动回收”时，燃油车行弱化并显示热损失；混动、增程、纯电行显示青色虚线反向流。
- 悬停节点显示一句解释，例如“逆变器：把电池直流电转换成电机需要的交流电，并控制电机扭矩”。
- 可用动效让箭头沿路径流动，但动效要慢，避免像游戏特效；建议 2.4s 线性循环，低透明度。

## 视觉规范

- 背景：`#F7F8FA`。
- 节点底色：白色，边框 `#D8DEE8`，8px 圆角。
- 燃油流：`#F59E0B`。
- 电能流：`#2563EB`。
- 机械流：`#475569`。
- 回收流：`#06B6D4`，虚线。
- 损失流：`#EF4444`，透明度 45%。
- 架构标签：深色文字 `#111827`，副文案 `#6B7280`。
- 每行高度桌面端 150-170px，部件节点宽 112-150px，箭头间距保持一致。
- 图标建议：油箱、发动机、齿轮箱、电池、闪电/逆变器、电机、车轮；不要使用复杂写实剖面图，保持“工程示意图”风格。

## 前端验收标准

1. 四行在同一坐标体系下对齐，用户能横向比较“谁多了什么、谁少了什么”。
2. 增程行不能出现发动机到车轮的机械直连。
3. 纯电行不能画传统多挡变速箱，只能画“单级减速器/差速器”。
4. 混动行必须体现发动机和电机都能参与驱动，以及电池双向流。
5. 制动回收状态下，燃油车与其余三种的差异必须明显。

参考依据：DOE AFDC 的汽油车、混动车、纯电车结构说明，以及 Toyota 对功率分流混动中发动机功率分配到车轮/发电机的说明。
