## 各岗位详解

### 📋 产品/项目管理

你在车企里的核心工作是**定义产品价值、推动跨团队协作、确保按时按质量交付**。

**为什么这些层最重要：**

| 层 | 你必须关心的理由 | 关键知识点 |
|----|:---|-----------|
| 🏢 车企语境 | 不懂 GVDP、APQP、节点交付物，你就没法推进项目 | 研发流程、V-model、里程碑/阀点、PPAP、8D |
| ⚡ 新能源 | 三电方案决定了整车成本、续航、充电体验 — 这是产品定义的核心 | 电池类型对比、电机特性、混动架构（增程vs插混）、800V高压 |
| 🧠 智能汽车 | 智驾等级和座舱功能是当下最核心的卖点，产品经理要能拆解技术对用户价值的影响 | ADAS分级、感知方案、域控制器、SOA/OTA |
| 🚗 整车认知 | 产品定义需要理解平台化思维、VIN、尺寸参数对用户的影响 | 汽车分类、驱动形式、平台化、VIN码 |
| 🔧 传统系统 | 你不需要设计发动机，但要能听懂工程语言 | 燃油/制动/转向的基本原理 |
| ⚙️ 机械基础 | 核心概念（扭矩/马力/传动比）是参数对比的基础 | 扭矩vs马力、变速箱类型 |

> 产品/项目管理岗位已有专属精简路径 → [产品/项目管理通识路径](./pm-path)

### 🛒 采购/供应链

你负责**评估供应商能力、谈判价格与交付、管理供应链风险**。

**为什么这些层最重要：**

| 层 | 你必须关心的理由 | 关键知识点 |
|----|:---|-----------|
| 🏢 车企语境 | 必须理解 Tier1/Tier2/OEM 分工 + APQP/PPAP 供应商批准流程 | 供应链角色、PPAP、APQP、IATF 16949 |
| ⚡ 新能源 | 电池是整车成本最高的部件，电机电控的供应链格局直接影响采购策略 | 电池材料与成本构成、电机类型、BMS |
| 🧠 智能汽车 | 芯片（Orin/8295/征程）和传感器的供应安全是近年最大风险 | 域控制器SoC、感知传感器供应链 |
| 🔧 传统系统 | 底盘件、制动系统仍是大宗采购项 | 悬架/制动/转向供应商格局 |
| 🚗 整车认知 | 平台化降低了零件定制比例→增加通用件采购杠杆 | 平台化概念、VIN码追溯 |
| ⚙️ 机械基础 | 核心参数用来审核供应商技术方案 | 扭矩/马力概念 |

> 采购/供应链岗位已有专属精简路径 → [采购/供应链通识路径](./procurement-supply-chain-path)

### 🏭 制造/质量

你负责**把设计图纸变成可批量生产的商品车，并确保每一台车在交付前符合设计标准和法规要求**：从来料检验、冲焊涂总四大工艺到售后反馈的完整质量闭环。

**为什么这些层最重要：**

| 层 | 你必须关心的理由 | 关键知识点 |
|----|:---|-----------|
| 🏢 车企语境 | APQP/PPAP/PFMEA/SPC/8D/IATF 16949 是制造和质量的共同工具箱 | 制造节点、FMEA（S×O×D）、CPK、8D |
| ⚡ 新能源 | 电池包装配工艺、高压安全、热失控防护是制造和质量的红线 | 电池PACK结构、电池安全标准（GB 38031）、BMS均衡 |
| 🔧 传统系统 | 底盘/动力总成合装是总装核心；发动机/制动是质量投诉高发区 | 发动机装配、制动异响/跑偏、排放失效 |
| 🚗 整车认知 | 车身结构决定焊接和涂装工艺；质量问题必须归类到正确系统 | 白车身、四大系统边界、平台化 |
| ⚙️ 机械基础 | 材料特性影响冲压焊接参数；材料疲劳/应力是根因分析基础 | 高强钢/铝合金成型特性、应力-应变 |
| 🧠 智能汽车 | 电子件装配/刷写/标定是总装新工序；功能安全和网络安全是质量新命题 | 域控制器装配、OTA初始化、ASIL分级、SOTIF |

> 制造/质量岗位已有专属精简路径 → [制造/质量通识路径](./manufacturing-quality-path)

### 🔬 测试/研发

你是技术最深入的角色：**设计子系统、制定验证计划、跑台架和实车试验**。

**为什么这些层最重要：**

| 层 | 你必须关心的理由 | 关键知识点 |
|----|:---|-----------|
| 🚗 整车认知 | 任何子系统测试都要放在整车环境里评估 | 四大系统、VIN码、驱动形式 |
| ⚙️ 机械基础 | 力学/热力学/流体/材料是所有工程计算的基础 | 牛顿定律、热效率、风阻、应力-应变 |
| 🔧 传统系统 | 发动机标定、制动调校、悬架K&C是测试研发的日常工作 | 发动机万有特性、制动效能、悬架动力学 |
| ⚡ 新能源 | 电机台架测试、电池循环寿命、BMS策略验证 | 电机MAP、电池SOH、电控策略 |
| 🧠 智能汽车 | 智驾系统测试（仿真+实车）是测试研发新战场 | 感知测试、决策验证、功能安全 |
| 🏢 车企语境 | DVP（设计验证计划）定义了你的测试任务 | V-model、DVP、APQP |

> 测试/研发岗位已有专属精简路径 → [测试/研发通识路径](./testing-rd-path)

## 为什么分岗位学

- **非汽车专业新人**最常见的困扰不是"知识点看不懂"，而是**"不知道哪些知识点跟我有关"**。
- 本站六层 54 个知识点是一张完整地图，但不同岗位只看其中一部分就够了。
- 岗位路径帮你**设定优先级**：核心层精读、重要层扫读、了解层知道有就行。

::: tip 通用建议
所有岗位的新人，建议**至少快速浏览第1层（整车认知）和第6层（车企工作语境）**。第1层建立全局观——遇到问题能归到正确系统；第6层是你在车企开会的入场券。
:::
