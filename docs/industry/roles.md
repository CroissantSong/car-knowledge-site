# 岗位与协作

## 研发组织架构

典型的主机厂研发中心组织：

```
                研发副总裁
                   │
    ┌──────┬────────┼────────┬──────┐
    │      │        │        │      │
  造型    工程     试制     试验   项目管理
(Design)(Eng.)  (Proto)  (Test)  (PM)
```

### 造型（Design/Studio）

- **外饰设计师**：整车外观造型
- **内饰设计师**：仪表台、座椅、门板设计
- **CMF 设计师**：Color（颜色）、Material（材料）、Finishing（表面处理）
- **数字模型师**：Alias/ICEM 曲面建模（A面）

### 工程（Engineering）

| 部门 | 负责系统 |
|------|----------|
| 车身工程 | 白车身、开闭件（车门/引擎盖/尾门） |
| 底盘工程 | 悬架、制动、转向、轮胎 |
| 动力总成 | 发动机、变速箱、混动系统 |
| 三电（新能源） | 电池、电机、电控 |
| 电子电气 | 线束、ECU、网络架构 |
| 智能驾驶 | 感知、决策、控制 |
| 智能座舱 | 中控/仪表、HMI、语音 |
| NVH | 噪音/振动/声振粗糙度 |
| 安全 | 碰撞安全、行人保护 |

### 试制（Prototype）

- VP/TT/PP 样车制造
- 试制车间管理
- 快速样件/3D 打印

### 试验（Test & Validation）

- **台架试验**：发动机台架、底盘测功机
- **整车试验**：三高（高温/高寒/高原）、耐久路试
- **碰撞试验**：正碰/侧碰/偏置碰
- **EMC 试验**：电磁兼容性

## 供应链角色

```
         OEM（主机厂）
            │
    ┌───────┼───────┐
    ▼       ▼       ▼
  Tier 1  Tier 1  Tier 1  ← 系统/模块级供应商
    │       │
    ▼       ▼
  Tier 2  Tier 2          ← 零部件级供应商
    │
    ▼
  Tier 3                   ← 原材料级供应商
```

| 层级 | 角色 | 提供 | 示例 |
|------|------|------|------|
| **OEM** | 主机厂 | 整车设计、总装、品牌 | 比亚迪、吉利、蔚来 |
| **Tier 1** | 一级供应商 | 系统/模块 | 博世（ESP/ECU）、宁德时代（电池） |
| **Tier 2** | 二级供应商 | 零部件 | 活塞供应商、传感器供应商 |
| **Tier 3** | 三级供应商 | 原材料/基础件 | 钢材、芯片裸片 |

## 项目管理与节点

### 里程碑（Milestone）

与 GVDP 关键节点对齐，每个里程碑都有明确的准入/准出标准。

### 阀点（Gateway）

重大决策评审点，通常需要高层审批才能通过。

> 阀点没过 → 项目不能进入下一阶段，类似于「门关着，必须把问题解决才能通过」。

### 核心交付物

每个节点需要提交一系列文档和数据：

- 设计数据（CAD 3D/2D）
- BOM（物料清单）
- DVP 报告（设计验证计划结果）
- DFMEA/PFMEA
- 成本分析
- 质量目标达成情况

### 项目经理（PM）

- 协调跨部门资源
- 跟踪项目进度和节点
- 管理风险和问题
- 向上汇报项目状态

## 新员工入门建议

1. **熟悉 GVDP**：了解整车开发从 0 到 1 的全过程
2. **掌握核心缩写**：SOP/EOP/DVP/BOM/PPAP 等是最常用的术语
3. **理解上下游**：你的工作交付给谁？接收谁的工作？
4. **多看实车**：拆车报告、对标分析是最好的学习材料
