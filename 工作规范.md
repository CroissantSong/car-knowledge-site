# 内容工作规范 · 汽车技术通识学习网站

> 全队（DeepSeek 工兵 / GPT 副手 / Opus 测试）在共享仓库协作时**必须遵守**，否则页面会出现乱码或图表错乱。

## 一、渲染硬性要求

1. **数学公式**：用 `$...$`（行内）/ `$$...$$`（独立）。VitePress 已开启 `markdown.math`（MathJax），会自动渲染。
   - ❌ 不要让 `\tau`、`\frac{}{}` 等以纯文本形式残留。
2. **图表：禁止使用 ASCII 手绘图**（`┌ └ │ ─ ┐` 拼出来的框图/曲线）——在网页上排版必乱、不美观。改用以下之一：
   - **Mermaid** 代码块（` ```mermaid `，需站点启用 mermaid 支持）；
   - **干净的 Markdown 表格**；
   - **内联 SVG**。
3. 任何改动后，必须 `npm run build` 通过，并 `npm run preview` **目视**确认公式与图表正常，才能提交。

## 二、标准协作流程（含图表/符号审核闭环）

1. **DeepSeek（工兵）**：写或完善知识点内容（文字、数据、结构、初版图示）。
2. **完成后必须交 GPT（副手）审一遍**：检查全部公式是否正确渲染、是否还有 ASCII 手绘图或符号问题。
3. **GPT 发现问题 → 先修好（重做图、修公式）→ 再 `git push`**。不允许带着乱码/坏图直接 push。
4. **Opus（测试）**：内容准确性与功能/构建最终验收。

## 三、仓库协作

- 共享仓库：`https://github.com/CroissantSong/car-knowledge-site`（已绑为 Multica 项目资源）。
- 流程：`multica repo checkout` → 修改 → `npm run build` 通过 → `git commit` → `git push origin main`。
